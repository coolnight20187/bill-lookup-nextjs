# Project Summary
The Bill Lookup System is a modern web application designed to streamline billing processes for businesses by effectively managing employee, customer, and transaction data. Built on a robust technology stack including Next.js, TypeScript, and PostgreSQL, the application enhances operational efficiency through real-time capabilities and an intuitive user interface. It serves as an essential tool for enterprises, enabling them to conduct billing transactions, manage inventory, and track sales seamlessly.

# Project Module Description
- **Database Initialization**: Sets up the PostgreSQL database structure for employees, customers, bills, and transaction history.
- **Authentication**: Implements user authentication with role-based access control, including login and logout functionalities.
- **Bill Lookup**: Provides an API for querying bills from multiple sources, including dual gateway systems.
- **Warehouse Management**: Manages the import and export of bills, along with inventory tracking.
- **Employee Management**: Offers tools for managing employee data, including full CRUD operations and work notes.
- **Customer Management**: Facilitates the management of customer details and sales transactions.
- **Transaction History**: Records and displays transaction history for auditing and reporting purposes.
- **Dynamic API Routes**: Handles various API requests for bill lookup, authentication, and warehouse operations.

# Technology Stack
- **Database**: PostgreSQL
- **Frontend**: Next.js 14, TypeScript, Tailwind CSS, shadcn/ui components
- **Authentication**: bcrypt for password hashing
- **Development Tools**: pnpm, ESLint

# Usage
1. **Install Dependencies**: Ensure Node.js and pnpm are installed.
2. **Clone the Repository**: Clone the project repository to your local machine.
3. **Install Packages**: Run the command to install necessary packages.
4. **Setup Environment Variables**: Create a `.env.local` file with the necessary environment variables.
5. **Initialize Database**: Run the database initialization script to set up the schema.
6. **Build the Project**: Execute the build command for your application.
7. **Run the Application**: Start the application using the appropriate command.
